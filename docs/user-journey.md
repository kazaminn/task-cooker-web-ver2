## 1. 世界観 — かざみんの酒場

ユーザーは「かざみんの酒場」の店主兼料理人。タスクは客からの注文であり、パンケーキが看板メニュー。
酒場は装飾レイヤーであり、型定義・ステータス名・遷移ロジックは一切変えない。

### 温度モデル

各ステータスは「温度」を持ち、視覚的な暖かさとアニメーションで表現する。

| ステータス | 温度感       | 色温度                 | アニメーション                         | アイコン挙動 |
| ---------- | ------------ | ---------------------- | -------------------------------------- | ------------ |
| order      | 静か・冷たい | slate blue (#cbd5e1)   | なし                                   | 静止         |
| prep       | 暖かい・準備 | amber/yellow (#fef08a) | なし                                   | 静止         |
| cook       | 熱い・pulse  | orange (#f97316)       | subtle pulse (reduced-motion 時は無効) | 微かな揺れ   |

> `prefers-reduced-motion: reduce` の場合、全アニメーションを無効化する。

## 3. User Journey Map

### Journey Stage 1: 初来店（First Visit）

| 項目               | 内容                                                     |
| ------------------ | -------------------------------------------------------- |
| トリガー           | URL アクセス or 紹介                                     |
| 関連ページ         | Landing (`/`), Login (`/login`), Signup (`/signup`)      |
| ユーザーアクション | Landing を見る → 「始める」→ Google OAuth でサインアップ |
| 感情状態           | 好奇心、期待。「面白そう」                               |
| ゲーミフィフック   | なし（まだ入店前）                                       |
| 酒場演出           | Landing に酒場の外観。「かざみんの酒場へようこそ」       |

### Journey Stage 2: 開店準備（Onboarding）

| 項目               | 内容                                                                                                  |
| ------------------ | ----------------------------------------------------------------------------------------------------- |
| トリガー           | 初回ログイン完了                                                                                      |
| 関連ページ         | Dashboard (`/home`)                                                                                   |
| ユーザーアクション | Personal Team 自動作成 → ツールチップツアー（3-4 ステップ、スキップ可）→ 最初のプロジェクト作成を促す |
| 感情状態           | ワクワク、少し不安。「何から始めよう」                                                                |
| ゲーミフィフック   | 初回ログインボーナス（+1 XP）                                                                         |
| 酒場演出           | 「厨房の案内をしますね」→ サイドバー・Quick Add・ステータスの説明                                     |

ツールチップツアーの内容:

1. **サイドバー**: 「ここがメニューボードです。プロジェクトを管理します」
2. **Quick Add**: 「カウンター越しに注文を受けましょう。ここに入力するだけ」
3. **ステータス**: 「注文 → 仕込み → 調理 → 提供。料理の工程でタスクを進めます」
4. **完了**: 「準備万端! 最初の注文を受けましょう」

### Journey Stage 3: 初めての注文（First Task）

| 項目               | 内容                                                                                 |
| ------------------ | ------------------------------------------------------------------------------------ |
| トリガー           | Quick Add or タスク作成ダイアログ                                                    |
| 関連ページ         | Dashboard (`/home`), Project Tasks (`/projects/:id/tasks`)                           |
| ユーザーアクション | タスクを作成 → ステータスを進める → 初めての serve                                   |
| 感情状態           | 達成感。「おお、できた!」                                                            |
| ゲーミフィフック   | タスク作成 +5 XP、ステータス進行 +3 XP/段階、serve +10 XP。初回 serve で特別トースト |
| 酒場演出           | serve 時に酒場風トースト: 「最初の一皿、お見事です! 🍽️」                             |

### Journey Stage 4: 日常営業（Daily Operations）

| 項目               | 内容                                                                                   |
| ------------------ | -------------------------------------------------------------------------------------- |
| トリガー           | 毎日のログイン                                                                         |
| 関連ページ         | Dashboard (`/home`), Project Tasks (`/projects/:id/tasks`), Task Detail (Right Drawer) |
| ユーザーアクション | Dashboard で状況確認 → タスク管理（DnD、フィルター）→ コメント追加 → serve             |
| 感情状態           | 習慣化、安定。「今日も営業するか」                                                     |
| ゲーミフィフック   | デイリーログイン +1 XP、ストリーク維持、草グラフが育つ                                 |
| 酒場演出           | 「おかえりなさい、店主。今日の注文は X 件です」                                        |

### Journey Stage 5: 繁盛店へ（Engagement & Mastery）

| 項目               | 内容                                                      |
| ------------------ | --------------------------------------------------------- |
| トリガー           | 継続利用（1週間〜）                                       |
| 関連ページ         | Profile (`/profile`), Dashboard (`/home`)                 |
| ユーザーアクション | 草グラフ確認 → レベルアップ → 称号確認 → ストリーク維持   |
| 感情状態           | 誇り、成長実感。「腕が上がってきた」                      |
| ゲーミフィフック   | レベルアップ通知、称号獲得、ストリーク記録更新            |
| 酒場演出           | レベルアップ時: 「腕を上げましたね! [称号名] に昇格です」 |

### Journey Stage 6: チーム営業（Team Use — Future）

| 項目               | 内容                                                         |
| ------------------ | ------------------------------------------------------------ |

## 4. ゲーミフィケーション設計

### データ保存

User ドキュメントにフィールドを追加:

```typescript
// users/{userId} に追加するフィールド
interface UserGamification {
  xp: number; // 累計 XP
  level: number; // 現在レベル
  title: string; // 現在の称号
  streak: number; // 現在の連続営業日数
  streakBest: number; // 最長連続営業日数
  lastActiveDate: string; // 最終アクティブ日 (YYYY-MM-DD)
}
```

草グラフ用サブコレクション:

```
users/{userId}/contributions/{YYYY-MM-DD}
  count: number            // その日のアクション数
  serveCount: number       // その日の serve 数
  date: string             // YYYY-MM-DD（クエリ用）
```

### XP ルール

| アクション              | XP  | 備考                                    |
| ----------------------- | --- | --------------------------------------- |
| タスク作成              | +5  |                                         |
| ステータス進行（1段階） | +3  | order→prep, prep→cook, cook→serve 各 +3 |
| serve（完了）           | +10 | ステータス進行の +3 とは別に加算        |
| コメント投稿            | +2  |                                         |
| プロジェクト作成        | +15 |                                         |
| デイリーログイン        | +1  | 1日1回のみ                              |

**設計方針:**

- 減点なし。モチベーションを下げない設計
- ステータスを戻しても XP は減らない（巻き戻しペナルティなし）
- 同じタスクを何度 serve しても XP は初回のみ（不正防止）
- XP 付与はクライアント側で計算し Firestore に書き込む（MVP）

### レベルシステム

| レベル | 称号             | 必要 XP | 酒場メタファー           |
| ------ | ---------------- | ------- | ------------------------ |
| Lv.1   | 見習い料理人     | 0       | 厨房に立ったばかり       |
| Lv.2   | 皿洗い卒業       | 30      | 基本を覚えた             |
| Lv.3   | 仕込み担当       | 80      | 準備ができるようになった |
| Lv.5   | 一人前の料理人   | 200     | 一通りこなせる           |
| Lv.7   | 腕利きの料理人   | 400     | 手際が良くなった         |
| Lv.10  | 繁盛店の店主     | 800     | 常連客がついた           |
| Lv.15  | 街の名店         | 1500    | 評判が広まった           |
| Lv.20  | 伝説の料理人     | 3000    | 語り継がれる存在         |
| Lv.25  | 美食の覇王       | 5000    | 頂点に立った             |
| Lv.30  | かざみんの後継者 | 8000    | 酒場の正統後継者         |

レベル計算式（参考実装）:

```typescript
function getLevel(xp: number): { level: number; title: string } {
  const levels = [
    { level: 30, xp: 8000, title: 'かざみんの後継者' },
    { level: 25, xp: 5000, title: '美食の覇王' },
    { level: 20, xp: 3000, title: '伝説の料理人' },
    { level: 15, xp: 1500, title: '街の名店' },
    { level: 10, xp: 800, title: '繁盛店の店主' },
    { level: 7, xp: 400, title: '腕利きの料理人' },
    { level: 5, xp: 200, title: '一人前の料理人' },
    { level: 3, xp: 80, title: '仕込み担当' },
    { level: 2, xp: 30, title: '皿洗い卒業' },
    { level: 1, xp: 0, title: '見習い料理人' },
  ];
  return levels.find((l) => xp >= l.xp) ?? levels[levels.length - 1];
}
```

### 草グラフ（パンケーキの焼き色）

GitHub の草グラフを「パンケーキの焼き色」で表現する。5段階。

| アクション数/日 | 段階           | 色名           | 視覚                        |
| --------------- | -------------- | -------------- | --------------------------- |
| 0               | empty          | 空セル         | グレー（--color-bg に近い） |
| 1-2             | light          | うっすら焼き色 | 薄いアンバー                |
| 3-5             | medium         | いい焼き色     | 中間アンバー                |
| 6-9             | well-done      | こんがり       | 濃いアンバー                |
| 10+             | perfect golden | 完璧な黄金色   | ゴールド                    |

アクション数のカウント対象:

- タスク作成
- ステータス変更
- コメント投稿
- プロジェクト作成
- serve 完了

表示場所: Dashboard サイドバー（Contrib セクション）、Profile ページ

### ストリーク（連続営業日数）

- 条件: 1日1 serve 以上で「営業日」としてカウント
- `lastActiveDate` と現在日を比較して連続判定
- 途切れた場合: streak を 0 にリセット。streakBest は保持
- 表示: Dashboard サイドバー下部に「連続営業 X 日目」
- 特別表記:
  - 7日連続: 「1週間連続営業!」
  - 30日連続: 「1ヶ月連続営業!」
  - 100日連続: 「100日連続! 伝説の始まり」

### 称号・実績バッジ

実績は条件達成時に付与。Profile ページに一覧表示。

| バッジ名         | 条件                        | 酒場メタファー     |
| ---------------- | --------------------------- | ------------------ |
| 初めての一皿     | 最初の serve                | 記念すべき初注文   |
| 10 皿達成        | serve 累計 10               | 常連客がつき始めた |
| 50 皿達成        | serve 累計 50               | 行列のできる店     |
| 早起き料理人     | 朝 6 時前にタスク作成       | 仕込みは朝が勝負   |
| 夜の料理人       | 深夜 0 時以降にタスク作成   | 深夜営業           |
| 7 日連続営業     | ストリーク 7 日             | 休まず営業         |
| 30 日連続営業    | ストリーク 30 日            | 鉄人               |
| メニュー開発者   | プロジェクト 5 個作成       | 多彩なメニュー     |
| おしゃべり料理人 | コメント累計 50             | 常連との会話       |
| スピード提供     | 作成から 1 時間以内に serve | 回転率の鬼         |

実績データの保存:

```
users/{userId}/achievements/{badgeId}
  unlockedAt: timestamp
  badgeId: string
```

## 5. 感情設計（Emotional Design）

### ステージごとの視覚的暖かさ

| Journey Journey Stage | 視覚トーン       | 暖かさ | 演出                             |
| --------------------- | ---------------- | ------ | -------------------------------- |
| 初来店                | 落ち着いた木目調 | 中     | 安心感。「入りやすい店構え」     |
| 開店準備              | 明るく案内的     | 中-高  | ツールチップの親しみやすさ       |
| 初めての注文          | 段階的に温かく   | 高     | serve 時のゴールデン演出         |
| 日常営業              | 安定した暖色     | 中     | 見慣れた厨房の安心感             |
| 繁盛店へ              | 達成感のゴールド | 高     | レベルアップ演出、草グラフの充実 |

### マイクロインタラクション

| インタラクション   | 演出                                        | reduced-motion 時              |
| ------------------ | ------------------------------------------- | ------------------------------ |
| serve 完了         | トースト通知 + 軽い confetti 的パーティクル | トースト通知のみ（アニメなし） |
| レベルアップ       | トースト + レベル数字のカウントアップ       | トースト通知のみ               |
| ストリーク更新     | サイドバーのストリーク数がバウンス          | 数字の即時更新のみ             |
| DnD ステータス遷移 | カードが滑らかに移動 + 列のハイライト       | 即時移動                       |
| 草グラフの日       | ホバーでツールチップ（日付 + アクション数） | 同じ                           |
| 称号獲得           | モーダル表示「新しい称号を獲得!」           | 同じ（アニメなし）             |

### コピートーン

酒場風だが、やりすぎない。機能的な情報伝達を最優先し、メタファーは補助的に添える。

| 場面                | メッセージ                                | トーン |
| ------------------- | ----------------------------------------- | ------ |
| タスク作成完了      | 「注文が入りました!」                     | 快活   |
| serve 完了          | 「お待たせしました!」                     | 達成感 |
| 全タスク serve 済み | 「本日の注文、全てお出ししました!」       | 祝福   |
| エラー（一般）      | 「おっと、少し手間取っています」          | 穏やか |
| ネットワークエラー  | 「厨房との接続が切れました。再接続中...」 | 説明的 |
| 削除確認            | 「この注文を取り消しますか?」             | 慎重   |

### 空状態メッセージ

| 画面                    | メッセージ                 | サブテキスト                       |
| ----------------------- | -------------------------- | ---------------------------------- |
| Dashboard（タスクなし） | 静かな厨房...              | 今日の注文を待っています           |
| プロジェクト一覧（空）  | メニューはまだ白紙です     | 最初のメニューを考案しましょう     |
| タスク一覧（空）        | 注文はまだ入っていません   | 最初の注文を受け付けましょう       |
