# レビューチェックリスト

PR を出す前に、以下の6ステップを**すべて**実行しろ。

## 手順

### 1. 対象ファイルを最初から最後まで通読する

- grep や find-and-replace だけで完了にするな
- 「文字列が N 回出現します」はレビューではない

### 2. 型定義との整合性を確認する

- `types/types.d.ts` の定義と実装が一致しているか
- ブランド型が正しく使われているか（`string` 生使いしていないか）
- `null` が混入していないか
- Firestore converter が正しく変換しているか

### 3. テストが存在しパスすることを確認する

- `pnpm test` を実行
- テスト件数を報告（0件なら PR を出すな）
- 新規機能には新規テストが含まれているか
- カバレッジが下がっていないか

### 4. 変更の影響範囲を列挙する

- この変更で影響を受けるファイル・コンポーネントを列挙
- データモデル変更なら Firestore ルールへの影響も確認

### 5. CLAUDE.md の禁止事項に違反していないか確認する

- `null` 使っていないか
- `as any` 使っていないか
- ファイルを4つ以上に分割していないか
- `src/components/` に置いていないか
- CSS に色の直値を書いていないか
- react-aria ではなく HTML5 API を使っていないか

### 6. 変更理由をコミットメッセージに書く

- commitlint 設定に従ったフォーマット
- why を書け。what だけ書くな

## 禁止事項

- **セルフレビュー禁止**: 自分のコードを自分でレビューするな。別モデル（Codex）か人間が行う
- **コスプレレビュー禁止**: Plan エージェントに「お前は Codex だ」と言わせるな
- **grep レビュー禁止**: 文字列検索だけで「整合性確認しました」と報告するな
- **目視レビュー禁止**: 「スクショで確認しました」はレビューとして認めない
