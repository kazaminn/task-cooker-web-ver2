## Semantic HTML

- Use `<button>`, `<nav>`, `<form>`, `<main>`, `<section>` etc. over generic `<div>` / `<span>`
- Use `react-aria-components` when it provides the needed interaction (Dialog, Menu, ComboBox, DnD, etc.)
- Do not use native HTML5 drag-and-drop — it lacks keyboard accessibility

## Keyboard Navigation

- All interactive elements must be fully operable with keyboard only
- Do not manipulate `tabindex` in ways that break natural tab order
- Dialogs and modals must trap focus and return focus on close
- Skip-nav landmarks: main, navigation, complementary

## ARIA

- Use `aria-expanded`, `aria-live`, `aria-label`, `role` when semantic HTML alone is insufficient
- Provide meaningful `alt` for informational images, empty `alt=""` for decorative ones
- Use `aria-label` for icon-only buttons

## Visual

- WCAG AA minimum contrast ratio (4.5:1 for normal text, 3:1 for large text)
- Text must be scalable — no fixed `px` for body text
- `prefers-reduced-motion` must be respected for animations (temperature pulse etc.)
- Status colors are theme-invariant for colorblind accessibility
